# ApplyPatch - Apply Git Unified Diff Patch

## Purpose
Apply a git-style unified diff patch to files. Used for precise, reviewable code changes.

## When to Use
- Applying changes generated by `preview_patch` tool
- Making surgical edits across multiple files
- Changes that need to be reviewable before applying
- Complex refactoring that touches multiple locations

## Usage
```
ApplyPatch{diff: "diff --git a/src/utils.py b/src/utils.py\n..."}
```

## Parameters
- `diff` (required): Git unified diff format patch content

## Returns
- Success message if patch applied cleanly
- Error message if patch failed (with rejection reason)

## Examples
```
# Apply a simple patch
ApplyPatch{diff: "diff --git a/src/utils.py b/src/utils.py\n--- a/src/utils.py\n+++ b/src/utils.py\n@@ -10,7 +10,7 @@ def calculate():\n-    return old_value\n+    return new_value\n"}
→ Patch applied successfully to src/utils.py

# Apply multi-file patch
ApplyPatch{diff: "diff --git a/src/a.py b/src/a.py\n...\ndiff --git a/src/b.py b/src/b.py\n..."}
→ Patch applied successfully to 2 files
```

## ⚠️ Critical Requirements
1. **Must read files first**: Always use `Read` tool to see current content before generating patch
2. **Git unified diff format**: Patch must be in standard git diff format
3. **Exact context lines**: Context lines in diff must match file content exactly
4. **Valid line numbers**: @@ hunk headers must have correct line numbers

## Failure Modes
| Symptom | Cause | Resolution |
|---------|-------|------------|
| "Patch does not apply" | Context lines don't match | Re-read file, regenerate patch with correct context |
| "Hunk FAILED" | Line numbers off | Check @@ hunk headers, verify line numbers |
| "File not found" | Wrong path in diff | Verify paths in diff match actual file paths |
| "Reversed patch detected" | Patch already applied | Check if changes already present in file |

## Diff Format
```diff
diff --git a/path/to/file.py b/path/to/file.py
--- a/path/to/file.py
+++ b/path/to/file.py
@@ -old_start,old_count +new_start,new_count @@
 context line
-removed line
+added line
 context line
```

## Notes
- Uses `git apply` under the hood
- Atomic: either all hunks apply or none do
- Prefer `Edit` tool for simple single-file changes
- Use `preview_patch` first to review patch before applying
- Works with any text file, not just code

## Related Tools
- `preview_patch` - Generate and review patch before applying
- `Edit` - Simple text replacement (non-diff)
- `Read` - Read file content before creating patch
