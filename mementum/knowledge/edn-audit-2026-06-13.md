# EDN Audit — Data Exchange Formats in OV5

Date: 2026-06-13
Scope: Runtime state, cross-daemon/pipeline data, persisted queues/metrics/decisions, and on-disk information exchange points in `~/.emacs.d`.

## Principle

OV5 should use **EDN** for all machine-to-machine data and info exchange. Markdown, plain text, JSON, JSONL, and ad-hoc sexp are tolerated only when:

- The artifact is primarily human-facing (reports, dashboards, logs).
- An external system requires the format (CI artifacts, RDF/Turtle).
- It is a scalar signal or lock file (PID, timestamp, heartbeat).

## High-priority migration candidates

1. **`var/tmp/experiments/<run-id>/results.tsv`**
   - Format: TSV with 5 schema versions (14, 20, 24, 27, 54 columns).
   - Consumers: experiment loop, self-heal, knowledge reasoning, pipeline statechart, shell scripts, Clojure world-store migration.
   - Plan: migrate to EDN sidecars and eventually the Datahike-backed World Store (`plans/ov5-world-store/`).

2. **`var/tmp/cron/auto-workflow-status.sexp`**
   - Format: Emacs sexp (`prin1`).
   - Consumers: cron shell scripts, watchdog, pipeline idle detector.
   - Plan: standardize on EDN; add a tiny shell/Elisp helper for shell consumers.

3. **`var/tmp/cross-subsystem-state.json`**
   - Format: JSON.
   - Carries: evolution hints, EMA history, verification results.
   - Plan: convert to EDN so Emacs and Clojure share one format.

4. **`var/tmp/researcher-controller.json`**
   - Format: JSON.
   - Consumers: strategic daemon, controller evolution.
   - Plan: EDN.

5. **`var/tmp/self-audit-result.el`**
   - Format: ad-hoc sexp, shell currently grep/perl-parses it.
   - Plan: EDN with a proper consumer, or at least JSON.

6. **`var/tmp/research-findings.md`**
   - Format: Markdown.
   - Keep Markdown for humans; add structured EDN sidecar for pipeline/evolution consumption.

7. **`var/context/<experiment-id>.sexp`**
   - Format: Emacs sexp.
   - Carries: business rationale, causal chain, decision rationale.
   - Plan: EDN.

8. **`var/tmp/checkpoints/active.ckpt`**
   - Format: JSON.
   - Carries: workflow recovery snapshot.
   - Plan: EDN.
