When extracting values from alist entries via `(cdr (assq key alist))`, a bare truthiness check `(if X (cdr X) default)` is unsafe — X could be a non-cons atom (e.g., symbol, number) causing `(cdr X)` to error. Even `(if (consp X) (cdr X) default)` is incomplete if `(cdr X)` might be nil or non-number and is later used in arithmetic comparisons like `(> value current-rate)`. The fully defensive pattern: `(if (and (consp X) (numberp (cdr X))) (cdr X) 0.0)`. This guards against both wrong-type-argument on `cdr` and on `>` comparison.
