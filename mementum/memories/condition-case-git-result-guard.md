In `gptel-auto-workflow--check-protected-configs`, `condition-case` only catches Elisp errors. `gptel-auto-workflow--git-result` returns `(output . exit-code)` WITHOUT throwing for non-zero exit codes. The `cond` must check both `nil` (from condition-case) AND non-zero exit codes (`(and (consp content) (/= 0 (cdr content)))`) to truly treat all unreadable configs as regressions. Old code returned `(cons "" 1)` on error, which silently passed the guard; new code returns `nil` and checks both failure modes.
