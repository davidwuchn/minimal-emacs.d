In `gptel-auto-workflow--check-protected-configs`, `default-directory` was bound directly to the result of `(gptel-auto-workflow--project-root)`. If that returns nil (e.g., outside a git repo), `default-directory` would become nil, causing all subsequent `git show` commands to fail with a directory error. Wrapped in `(or proj-root default-directory)` so that the current buffer's directory is preserved as a fallback — graceful degradation instead of hard error.
