### ReVeal: Self-Verification via Multi-Turn RL
**Source**: https://openreview.net/forum?id=q56ZI1Co43

**Pattern**: Widen verification-generation asymmetry. Model alternates between generating code AND generating tests to verify it. TAPO handles credit assignment across turns. This enables 20+ self-improvement turns despite training on only 3.

**How it works**: 
1. Generate candidate code
2. Generate test cases for that code
3. Execute tests to verify
4. Use test feedback for next iteration
5. Credit assignment propagates backward through turn sequence

**Apply to Emacs**: Implement `gptel-self-verify` loop that:
```elisp
(defun gptel--self-verify (code)
  "Generate code, tests, verify, iterate."
  (let* ((candidate (gptel--generate code))
         (tests (gptel--generate-tests candidate))
         (result (gptel--execute-tests tests)))
    (if (gptel--pass-p result)
        candidate
      (gptel--self-verify (gptel--incorporate-feedback candidate tests result)))))
```
