In gptel-auto-workflow-ontology-router.el, `gptel-auto-workflow--check-all-targets-consistency` calls `gptel-auto-workflow--cross-backend-consistency` inside a maphash lambda without error handling. If `parse-all-results` returns corrupt data or the consistency check throws, it crashes the entire maphash iteration. Guard added: `(condition-case nil ... (error nil))` so nil check becomes `nil` and existing `(or (plist-get check :backend-count) 0)` handles it gracefully.
