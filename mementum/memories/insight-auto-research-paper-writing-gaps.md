# Auto-research paper writing gaps

Studied auto_research paper writing skill (https://victorchen96.github.io/auto_research/skill/paper-writing.html). Key insight: Auto-research models scientific paper writing as hierarchical skill group with 5 sub-skills, 4 quality gates, and iterative review loops, producing 8.5/10 survey papers autonomously.

## Five sub-skills
1. **Literature Survey** (20% time) — 4-stage pipeline: Recall → LQS Score → Classify → Upgrade
2. **Paper Structure & Logic** (35% time) — Chapter architecture, paragraph logic, taxonomy design
3. **Experiment Design** (20% time) — 4-stage loop: Design → Execute → Iterate → Report
4. **Academic Figures & Tables** (10% time) — High information-density tables and vector figures
5. **Peer Review Simulation** (15% time) — Multi-persona scoring with weakness routing

## Highest-leverage gaps for OV5
1. **Structured literature survey** — OV5 does ad-hoc research; auto-research has 4-stage pipeline with LQS scoring
2. **Research quality gates** — OV5 has 7 gates for code but none for research; auto-research has 5 gates
3. **Peer review simulation** — OV5 has single-grader; auto-research has 5 reviewer personas with weakness routing
4. **Iterative improvement loop** — OV5 doesn't track score progression; auto-research progresses 6.0 → 8.5+
5. **Hypothesis pre-registration** — OV5 experiments lack statistical planning; auto-research requires pre-registration

## Strategic insight
Auto-research optimizes **research quality** through structured pipelines and peer review. OV5 optimizes **code improvement** through experiment loops and self-healing. Integration: add auto-research's research pipeline to OV5's code improvement loop for both research rigor and code quality.

## Implementation priority
Start with structured literature survey (foundation), then research quality gates (standards), then peer review simulation (iteration driver).
